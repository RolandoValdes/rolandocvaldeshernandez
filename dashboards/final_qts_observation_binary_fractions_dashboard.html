Topic: Binary Fractions in Python | Class: Year 10 Computer Science | Duration: 2 x 45 minutes (90 minutes)

Learning Objective: Convert decimal numbers with fractional parts into binary and explain how Python can automate the conversion process.

Observation Overview

Observation type: Final AO QTS Observation

Success criteria:

  • Identify binary fractional place values such as 1/2, 1/4, 1/8 and 1/16.
  • Convert simple decimal fractions and mixed denary numbers into binary manually.
  • Use repeated division and repeated multiplication to convert whole and fractional parts.
  • Trace, interpret and modify Python code that converts decimal numbers into binary.

Prior learning: Denary and binary whole number place values, Basic denary-to-binary conversion, Variables and simple Python input/output, Sequence and iteration in Python

Common misconceptions:

Thinking binary fractions use tenths, hundredths and thousandthsReading repeated-division remainders from top to bottomRecording the decimal result instead of the integer part during repeated multiplicationForgetting to combine the whole and fractional binary parts with a binary pointAssuming every decimal fraction can be represented exactly in binary

Likely Observation / Professional Dialogue Questions

Recorded lesson

Why did you choose this lesson for your final observation?

30-second answer

It shows a clear transition into GCSE content and provides evidence of subject knowledge, modelling, assessment, adaptive teaching and programming links.

Detailed answer

I chose this lesson because binary fractions are demanding but accessible when carefully scaffolded. It allows me to demonstrate how I move pupils from prior knowledge of whole-number binary into new GCSE data representation content, then connect that knowledge to Python algorithms.

Planning

How does the lesson build on prior learning?

30-second answer

It builds on whole-number binary, place value, basic Python variables, input and iteration.

Detailed answer

Students already know that computers use binary and have encountered whole-number place values. This lesson extends that by introducing fractional place values to the right of the binary point and then links the manual method to Python loops and variables.

Subject knowledge

What are the key concepts pupils need to understand?

30-second answer

They need to understand the binary point, negative powers of two, repeated division, repeated multiplication and why precision matters.

Detailed answer

The central concepts are that binary fractions use negative powers of two, the whole and fractional parts are converted using different methods, and Python can automate this algorithm. The precision discussion introduces pupils to the idea that not all denary fractions have exact binary representations.

Assessment

How did you check that pupils understood the conversion process?

30-second answer

I used questioning, mini-whiteboards, worked examples, differentiated practice, live checking and an exit ticket.

Detailed answer

I checked understanding at several points: during the starter, when pupils identified fractional place values, during manual conversion practice, while they traced Python code, and at the end through the exit ticket. This helped identify whether errors came from place value, method or programming syntax.

Adaptive teaching

How did you support pupils who struggled?

30-second answer

I used visual place-value charts, chunked examples, repeated modelling and Bronze tasks with simple fractions.

Detailed answer

Support was built in through visual models, step-by-step worked examples and simpler Bronze tasks such as 5.5 and 0.75. I also separated the whole-number and fractional steps so pupils did not have to process the entire algorithm at once.

Challenge

How did you stretch higher-attaining pupils?

30-second answer

They converted more complex values, traced Python logic and explored binary precision limits.

Detailed answer

Higher-attaining pupils moved to Gold conversions such as 13.1875, then tested the Python code with additional values. They were also challenged to explain why values such as 0.1 may not be stored exactly in binary.

Behaviour

How did you maintain focus during a technical lesson?

30-second answer

Through clear routines, short teacher input, frequent checks, structured tasks and active circulation.

Detailed answer

Because the content is conceptually demanding, I maintained focus by chunking the lesson into short stages, using mini-whiteboards and paired discussion, and circulating during independent and programming tasks to keep pupils accountable.

Reflection

What would you improve next time?

30-second answer

I would add an earlier diagnostic question to separate mathematical misconceptions from Python misconceptions.

Detailed answer

Next time I would include a quick hinge question before moving into Python, so I could confirm that pupils understand manual conversion before adding programming complexity. This would make the transition from data representation to coding even sharper.

Safeguarding

How does the lesson promote responsible computing?

30-second answer

It helps pupils understand that digital systems represent numbers in ways that can affect accuracy and decision-making.

Detailed answer

The lesson introduces pupils to the fact that computers represent values using finite binary patterns. This supports responsible computing because pupils begin to understand that software output depends on representation, precision and algorithm design.

Curriculum

Why is this appropriate as an introduction to GCSE content?

30-second answer

It links KS3 binary knowledge to GCSE data representation and programming in a structured, accessible way.

Detailed answer

The lesson is appropriate because Year 10 pupils can access it through prior binary knowledge, but it also raises the level of challenge by introducing binary fractions, algorithms and Python implementation. This gives them a strong bridge into GCSE Computer Science.
